PHASE 2 : MODÈLE DE DONNÉES (ENTITIES)
═══════════════════════════════════════════════════════════════════════════════

[ ] 2.1 Créer l'entité User
    - id (String, PK)
    - login (String, unique)
    - password (String, encrypted)
    - roles (Enum: ADMIN, FORMATEUR, ETUDIANT)
    - Relations : OneToOne avec Etudiant

[ ] 2.2 Créer l'entité Etudiant
    - id (String, PK)
    - matricule (String, unique)
    - nom (String)
    - prenom (String)
    - email (String, unique)
    - dateInscription (Date)
    - Relations :
      * OneToOne avec User
      * OneToMany avec Inscription
      * OneToMany avec Note

[ ] 2.3 Créer l'entité Formateur
    - id (String, PK)
    - nom (String)
    - specialite (String)
    - email (String, unique)
    - Relations :
      * OneToMany avec Cours
      * OneToMany avec Seance

[ ] 2.4 Créer l'entité Session
    - id (String, PK)
    - semestre (String)
    - anneeScolaire (String)
    - Relations : OneToMany avec Cours

[ ] 2.5 Créer l'entité Cours
    - code (String, PK)
    - titre (String)
    - description (String)
    - formateurId (FK vers Formateur)
    - sessionId (FK vers Session)
    - Relations :
      * ManyToOne avec Formateur
      * ManyToOne avec Session
      * OneToMany avec Inscription
      * OneToMany avec Note
      * ManyToMany avec Groupe (via cours_groupe)
      * OneToMany avec Seance

[ ] 2.6 Créer l'entité Groupe
    - id (String, PK)
    - nom (String)
    - Relations : ManyToMany avec Cours (via cours_groupe)

[ ] 2.7 Créer l'entité CoursGroupe (Junction Table)
    - id (String, PK)
    - groupeId (FK vers Groupe)
    - coursCode (FK vers Cours)

[ ] 2.8 Créer l'entité Inscription
    - id (String, PK)
    - dateInscription (Date)
    - status (String)
    - etudiantId (FK vers Etudiant)
    - coursCode (FK vers Cours)
    - Relations :
      * ManyToOne avec Etudiant
      * ManyToOne avec Cours

[ ] 2.9 Créer l'entité Seance
    - id (String, PK)
    - date (Date)
    - heure (Time)
    - salle (String)
    - coursCode (FK vers Cours)
    - formateurId (FK vers Formateur)
    - Relations :
      * ManyToOne avec Cours
      * ManyToOne avec Formateur

[ ] 2.10 Créer l'entité Note
    - id (String, PK)
    - valeur (Float)
    - dateSaisie (Date)
    - etudiantId (FK vers Etudiant)
    - coursCode (FK vers Cours)
    - Relations :
      * ManyToOne avec Etudiant
      * ManyToOne avec Cours

[ ] 2.11 Configurer les annotations JPA
    - @Entity, @Table
    - @Id, @GeneratedValue
    - @Column (contraintes, unique, nullable)
    - @OneToOne, @OneToMany, @ManyToOne, @ManyToMany
    - @JoinColumn, @JoinTable
    - Cascade types appropriés

[ ] 2.12 Créer les DTOs si nécessaire
    - DTOs pour les requêtes API
    - DTOs pour les réponses API
    - Mappers (MapStruct ou manuels)
